## 1. 개인 역할 분담 및 실행

본 프로젝트는 기존의 파트별 단독 개발 방식에서 벗어나 **페어 프로그래밍과 기능 중심 협업 방식**으로 진행한다.

### 주영찬 -팀장 & 박재혁 — Core Driving & AI Duo

선두 차량과 후미 차량의 물리적 제어 및 자율주행 알고리즘을 공동 개발한다.

- 선두 차량 A의 카메라 기반 CNN 라인 트레이싱 구현
- 선두 차량의 차선 변경 판단 및 주행 로직 구현
- 후미 차량 B, C의 YOLO 기반 전방 차량 및 장애물 인식
- 360도 LiDAR를 활용한 차량 간 거리 측정 및 차간거리 유지 제어
- 모터 속도와 조향각 데이터를 추출하여 백엔드로 전송
- 선두 차량과 후미 차량의 주행 데이터 동기화
- 실제 주행 시험을 통한 인식 모델 및 제어 게인 조정

### 이명연 — Hardware UI & Dashboard Co-development

차량의 시각적 상태 표시 기능을 담당하고, 팀장과 함께 통합 대시보드를 개발한다.

- 급감속 시 브레이크등 제어
- 차선 변경 시 방향지시등 제어
- 비상 정지 시 비상 점멸등 제어
- React 기반 관제 대시보드 화면 설계
- 차량별 상태, 센서값, 연결 상태 및 시스템 로그 표시
- 실시간 차간거리 그래프와 차량 상태 컴포넌트 제작
- PyWebView 환경에서 React 화면이 실행되도록 데스크톱 UI 구성

### 권신용 / Backend·통신 인프라 & Dashboard Co-development

전체 시스템과 데이터 흐름을 총괄하고, 차량 간 통신 및 대시보드 백엔드를 구축한다.

- 전체 일정 관리 및 기능 통합
- MQTT 브로커 설정 및 차량 간 V2V 통신 구성
- FastAPI 기반 통합 백엔드 서버 구축
- tmux 세션 기반 차량별 프로세스 실행 환경 구성
- 차량의 속도, 조향각, 차간거리 및 이벤트 데이터 수신

- 장애물 인식 및 차선 변경 이벤트를 TTS 출력과 연동

## 2. 과제 목표

여러 대의 한백전자 AutoCar Prime을 선두 차량과 후미 차량으로 구성하여 일정한 대열과 차간거리를 유지하는 자율 군집주행 시스템을 구현한다.

선두 차량 A는 카메라 기반 CNN 라인 트레이싱을 이용하여 주행 경로를 추종하고, 필요할 경우 차선 변경을 수행한다. 후미 차량 B와 C는 YOLO 기반 객체 인식과 360도 LiDAR 거리 정보를 이용하여 바로 앞 차량을 추종하고 목표 차간거리를 유지한다.

각 차량의 NVIDIA 기반 SODA 장치에서는 영상 인식, LiDAR 처리, 모터 및 조향 제어를 독립적으로 수행한다. 차량의 속도, 조향각, 차간거리, 인식 결과 및 동작 상태는 MQTT를 통해 공유한다.

노트북에서는 React 화면을 PyWebView로 실행하는 통합 관제 대시보드를 제작한다. 대시보드에서는 여러 RC카의 상태를 한 화면에서 확인하고 차량별 제어, 전체 군집주행 시작·정지, 긴급 정지 및 시스템 로그 확인 기능을 제공한다.

## 3. 수행 계획서

### 3.1 차량별 기본 기능 점검

- AutoCar Prime별 카메라, 모터, 조향, LED 및 LiDAR 동작 상태 확인
- 차량별 IP 주소와 차량 ID 설정
- 차량 A는 선두 차량, 차량 B와 C는 후미 차량으로 역할 구분
- 각 차량 IP에 연결된 Jupyter Notebook에서 센서와 구동 기능 개별 시험
- 최종 실행 전 Notebook 제어 프로그램을 종료하고 통합 실행 프로그램만 운용

### 3.2 선두 차량 자율주행 구현

- 카메라 영상 수집 및 전처리
- CNN 기반 라인 트레이싱 모델 적용
- 주행선 중심 오차를 이용한 조향 제어
- 차선 변경 가능 구간과 장애물 상태를 고려한 차선 변경 로직 구현
- 현재 속도, 조향각, 차선 상태 및 이벤트 정보를 MQTT로 전송

### 3.3 후미 차량 군집 추종 구현

- YOLO를 이용하여 전방 차량과 장애물 인식
- 360도 LiDAR로 앞 차량과의 실제 거리 측정
- 영상 기반 방향 정보와 LiDAR 거리 정보를 결합
- 목표 차간거리에 따라 모터 속도 제어
- 앞 차량의 이동 방향에 따라 조향각 제어
- 전방 차량 소실 또는 최소 안전거리 진입 시 자동 정지

### 3.4 차량 간 통신 및 데이터 동기화

- 차량별 SODA에서 MQTT client 실행
- 차량 ID, 속도, 조향각, 차간거리, 주행 모드 및 이벤트 정보 송수신
- 선두 차량의 감속, 정지 및 차선 변경 정보를 후미 차량에 전달
- heartbeat를 이용하여 차량 연결 상태 확인
- 통신 두절 시 해당 차량의 자동 감속 및 정지
- FastAPI 백엔드에서 모든 차량의 텔레메트리 데이터 수집

### 3.5 PyWebView 통합 대시보드 제작

- React 기반 프론트엔드 화면 제작
- React 화면을 PyWebView 데스크톱 창으로 실행
- FastAPI와 WebSocket을 이용한 실시간 데이터 연동
- 차량 A, B, C의 연결 상태와 주행 모드 표시
- 차량별 속도, 조향각, 차간거리 및 객체 인식 상태 표시
- 실시간 차간거리 그래프와 시스템 로그 표시
- 차량별 자동주행 시작·정지 기능 구현
- 전체 군집주행 시작, 전체 정지 및 긴급 정지 기능 구현
- 선택 차량의 카메라 영상 확인 기능 구현

### 3.6 LED 및 TTS 상태 알림 구현

- 급감속 시 브레이크등 점등
- 차선 변경 방향에 따라 방향지시등 점멸
- 비상 정지 시 양쪽 비상 점멸등 작동
- 장애물 인식, 차선 변경 및 비상 정지 이벤트를 TTS로 출력
- 차량 내부 상태와 대시보드 상태가 일치하도록 이벤트 데이터 연동

### 3.7 프로세스 운용 및 시스템 점검

- tmux의 session, window, pane을 이용하여 제어, 영상, 통신 및 로그 프로세스 모니터링
- 임시 시험은 background process와 nohup을 이용

- kill, killall, pkill을 이용한 비정상 프로세스 종료
- free, df, du를 이용한 메모리 및 저장공간 확인
- iotop, iftop을 이용한 입출력 및 네트워크 부하 확인
- netstat, ss를 이용한 MQTT, FastAPI 및 WebSocket 포트 상태 확인

### 3.8 통합 시험 및 결과 정리

- 차량 B, C의 YOLO 및 LiDAR 거리 측정 시험

## 4. 시행 목적

본 프로젝트는 한 대의 RC카가 독립적으로 주행하는 자율주행 실습을 넘어, 여러 대의 차량이 통신을 통해 주행 정보를 공유하고 일정한 차간거리와 대열을 유지하는 협력형 자율주행 시스템을 구현하는 것을 목적으로 한다.

선두 차량에는 CNN 기반 라인 트레이싱과 차선 변경 기능을 적용하고, 후미 차량에는 YOLO 객체 인식과 360도 LiDAR 기반 거리 유지 기능을 적용한다. 이를 통해 영상 인식과 거리 센서를 결합한 군집 추종 제어를 구현한다.

또한 MQTT, FastAPI, WebSocket 및 PyWebView 대시보드를 이용하여 여러 차량의 데이터를 실시간으로 수집하고 통합 관리한다. LED와 TTS를 통해 차량의 감속, 차선 변경, 장애물 및 비상 정지 상태를 시각적·음성적으로 전달한다.

프로젝트 수행 과정에서 자율주행 알고리즘, 차량 제어, V2V 통신, 실시간 대시보드, Linux 프로세스 관리, 장애 대응 및 GitHub 기반 협업 능력을 종합적으로 향상시키고자 한다.
